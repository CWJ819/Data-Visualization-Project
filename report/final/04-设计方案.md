# 04 · 设计方案

对应系统文档要求：设计需求、设计介绍、案例展示、AI 使用声明。

---

## 1. 设计需求

围绕核心论点"文人精神世界由外向转内向"，系统需满足：

1. **叙事性**：以时间为横轴，把情感、意象、体裁、地理、词汇五个维度组织成一条可被引导阅读的故事线（盛唐 → 动乱 → 晚唐 → 宋初），而非零散图表堆叠。
2. **多视图协调**：各视图共享同一时间轴语义，支持 **brushing & linking**——在时间轴上选定一个时期，其余视图同步聚焦该时期。
3. **细节按需展开（details-on-demand）**：宏观指标（情感分、externality）可下钻到具体诗作，保证代理指标可追溯回原文。
4. **对比性**：支持"唐 vs 宋"的纵向并置（双词云、同题异调对照）。
5. **数据诚实**：低置信/低样本数据需在视觉上弱化或标注，不把 LLM 猜测或默认年份包装成事实。
6. **可复现、端到端可运行**：静态数据 + 纯前端即可启动，便于现场演示与评分复现。

## 2. 设计介绍

### 2.1 技术栈

**React + Vite + ECharts**（前端，无后端；数据为静态 JSON）。
选型理由、依赖清单、目录结构、数据契约与三人分工详见 `../../memory/design/frontend_stack_design.md`。要点：

- 规划中的图表（themeRiver / 地图 / 词云 / 堆叠面积 / 关系图）几乎都是 ECharts **原生类型**，开箱即用，把精力留给叙事与数据解读。
- 数据已是静态 JSON，`fetch` 即用，`vite build` 出纯静态包即可交付演示。
- 组件化便于三人并行开发，git 低冲突。

### 2.2 六屏叙事结构

| 屏 | 主题 | 子任务 | 主图 | 视觉通道 | 交互 |
|:--|:--|:--|:--|:--|:--|
| S1 | 历史时间轴 + 情感温度计 | T1 | 折线 + 事件标注带 | 位置=年份/情感；红色警示带=755 断裂 | 拖动时间轴 → 联动其余屏 |
| S2 | 意象的生死流转 | T2 | 意象河流图 themeRiver | 河宽=频率；色=宏大/私人；叠加 externality 曲线 | 点选意象 → 高亮其时间轨迹 |
| S3 | 诗体的新陈代谢 | T3 | 百分比堆叠面积 | 面积=体裁占比 | hover → 该时期各体裁明细 |
| S4 | 地域空间坍缩 | T4 | 中国地图 + 散点/热力 | 点大小/热度=地名频次 | 切换时期 → 看版图收缩 |
| S5 | 唐诗 vs 宋词 | T5 | 双色对比词云 | 色=唐/宋/公共；字号=频率 | 点词 → 下钻例句（details-on-demand） |
| S6 | 两大断裂断面 | T1/T2 | before/after 双柱 + rising/falling 词 + 策展星 | 色差=变化方向；柱高=绝对值 | 点策展星 → 同 S1/S2 的诗作 drawer |

### 2.3 多视图协调

#### 联动架构

全局共享一个 `selectedPeriod` 状态（Zustand store），由任意视图写入、所有视图订阅。

**① 时期轴联动（S1 ↔ S2 ↔ S3 ↔ S4）**

S1/S2/S3 的 X 轴均为同一批有序时期。用户 hover 或点击某时期列时：

- S1 该列折线打点高亮
- S2 对应时期的河流段变亮，其余半透
- S3 对应面积列高亮
- S4 地图切换为 `place_geo.by_period[selectedPeriod]` 的散点
- 繁星叠加层过滤为该时期星点

**② 事件标注 → S6 下钻**

S1/S2 的 755、1127 竖线（`markLine`）点击后，页面滚动至 S6 并高亮对应断面板块，`curated_stars` 列表自动展开。这是唯一需要跨屏滚动的联动。

**③ 繁星 → 诗作详情 drawer**

S1/S2 叠加层的星点与 S6 的 `curated_stars` 条目共用同一个侧边 drawer，点击后展示：

```text
key_line（名句）  |  theme_llm + types_llm（LLM标注）
polarity_llm（极性颜色印证）  |  stage_llm（创作背景）
```

星点颜色编码：积极 → 金色，消极 → 深蓝，中性 → 灰，LLM未覆盖 → 白边空心。

**④ 情感类型筛选（可选增强）**

繁星层提供 legend 勾选框（豪迈 / 离愁 / 思乡 / 闲适 / 家国 / 爱情 / 哲思 / 孤独），勾选后全屏星点按 `types_llm` 过滤，不影响流图本体。

**⑤ S5 词云（孤立，单向）**

词云为全量唐/宋对比，无时期维度，不接入 `selectedPeriod`。切换时期时词云保持不动，添加一行文字提示："当前词云为全唐/全宋总体，非所选时期专属。"

#### 联动简图

```text
meta.json (period_order / events)
        │
        ▼
S1(情感曲线) ──┬── selectedPeriod ──► S4(地图)
S2(意象河流) ──┤         │
S3(体裁演化) ──┘         └──► 繁星层过滤
                               │
S1/S2 markLine(755/1127) ──────┼──► S6(断裂断面) 滚动高亮
                               │
stars.json ─── 叠加星点 ───────┴──► 点击 drawer
                S6 curated_stars ────────────────────┘（同一 drawer）

S5(词云) ── 孤立，不驱动其他视图
```

- **案例 A · 情感断崖（T1）**：在 755 年红色警示带前后，情感均值曲线应出现可见落差——盛唐高位 → 动乱骤降。届时给出截图 + 下钻到杜甫安史之乱诗作的操作路径。

- **案例 C · 版图坍缩（T4）**：地图从盛唐的塞北/西域密集，收缩到宋初的汴京/江南都市圈，直观呈现"帝国心灵版图的萎缩"。

## 4. AI 使用声明

本项目在以下环节使用了 AI，并对其输出保持人工审查：

### 4.1 DeepSeek（大语言模型，作为数据标注器）

- **用途**：对诗词文本做**情感标注（二维：极性 × 类型）、题材辅助分类、意象抽取、创作阶段推断**，产物为 `pipeline/output/enrich/enriched.jsonl`，由 `metrics.py` 按 `id` 自动关联进 `dashboard/data/`。
- **设计原则与边界**：
  - 明确**不**用 LLM 推断精确创作年份（易幻觉），改为"阶段 + 置信度"。
  - 题材分类仅作辅助维度，不作主结论。
  - 引入 **schema 校验、batch 返回 ID 完整性校验、一致性检验**（同诗多次调用比对）以约束可靠性。
- **已知局限**：当前抽样一致性偏低（题材 50% / 意象 10%），相关字段**尚未作为正式结论**，待改进 prompt 后重跑。

### 4.2 Claude（Claude Code，作为开发与文档协作）

- **用途**：协助数据流水线的代码审查与修复（API key 去硬编码、断点续跑、繁简/默认年份等数据质量问题诊断）、以及本报告与 `memory/` 文档的结构整理与内容整合。
- **人工把关**：所有数据清洗结论均由 `pipeline/qa.py` 复跑校验；设计决策与文学论点由小组成员最终判定与负责。

### 4.3 总体原则

AI 用于**加速标注与工程**，不替代研究判断。所有代理指标（情感分、externality、阶段推断）在系统中均保留下钻回原诗的路径，确保结论可追溯、可质疑、可复现。
